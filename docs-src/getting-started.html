Getting Started

Get up and running with pm7-ui - The First UI Library Built for AI Coding Agents.

Installation

pm7-ui provides a single package that works with ALL frameworks:

NPM Installation

npm install @pm7/core

CDN Installation

<link rel="stylesheet" href="https://unpkg.com/@pm7/core@2.0.0/dist/pm7.css">
<script src="https://unpkg.com/@pm7/core@2.0.0/dist/pm7.js"></script>

That's it! One package works with React, Vue, Angular, Svelte, or vanilla HTML.

Basic Usage

Step 1: Import the CSS

<!-- In your HTML -->
<link rel="stylesheet" href="node_modules/@pm7/core/dist/pm7.css">

// Or in JavaScript (React, Vue, etc.)
import '@pm7/core/dist/pm7.css';

Step 2: Use PM7 Components

PM7 components work with simple CSS classes in ANY framework:

Vanilla HTML

<button class="pm7-button pm7-button--primary">
  Click me
</button>

<div class="pm7-card">
  <div class="pm7-card-header">Card Title</div>
  <div class="pm7-card-body">Card content goes here.</div>
</div>

React

function App() {
  return (
    <div>
      <button className="pm7-button pm7-button--primary" onClick={() => alert('Clicked!')}>
        Click me
      </button>
      
      <div className="pm7-card">
        <div className="pm7-card-header">Card Title</div>
        <div className="pm7-card-body">Card content goes here.</div>
      </div>
    </div>
  );
}

Vue

<template>
  <div>
    <button class="pm7-button pm7-button--primary" @click="handleClick">
      Click me
    </button>
    
    <div class="pm7-card">
      <div class="pm7-card-header">Card Title</div>
      <div class="pm7-card-body">Card content goes here.</div>
    </div>
  </div>
</template>

Step 3: Interactive Components (Optional)

Interactive components like menus and dialogs auto-initialize. For manual control:

<script type="module">
  import '@pm7/core';
  
  // Everything auto-initializes!
  // Or manually initialize after adding dynamic content:
  PM7.init();
</script>

TypeScript Support

pm7-ui includes full TypeScript support with type definitions for all components.

Setup

TypeScript types are automatically included when you install @pm7/core. No additional packages needed!

Using Component Instances

Create and use component instances:

// Access tab selector instance
const tabElement = document.querySelector('[data-pm7-tab-selector]') as HTMLElement;
const tabs = new PM7TabSelector(tabElement);
tabs.selectTabById('settings-tab');
const activeTab = tabs.getActiveTab();

// Menu instance
const menuElement = document.querySelector('[data-pm7-menu]') as HTMLElement;
const menu = new PM7Menu(menuElement);
menu.open();

// Tooltip instance
const tooltipElement = document.querySelector('.pm7-tooltip') as HTMLElement;
const tooltip = new PM7Tooltip(tooltipElement);
tooltip.updatePosition();

Typed Utilities

import { showToast, confirm, createDialog } from '@pm7/core';

// Fully typed toast options
const toast = showToast({
  message: 'Operation successful',
  type: 'success',
  duration: 3000,
  position: 'top-right'
});

// Async confirmation dialogs
const userConfirmed = await confirm('Delete this item?');
if (userConfirmed) {
  // Proceed with deletion
}

See the TypeScript documentation for complete API reference and examples.

Customization

pm7-ui uses CSS custom properties (variables) for theming. You can customize colors, spacing, and more:

Override CSS Variables

:root {
  /* Custom primary color */
  --pm7-primary: #your-color;
  --pm7-primary-hover: #your-hover-color;
  
  /* Custom spacing */
  --pm7-spacing-unit: 0.25rem;
  
  /* Custom fonts */
  --pm7-font-sans: 'Your Font', sans-serif;
}

Dark Mode

pm7-ui includes comprehensive dark mode support with automatic theme detection and a built-in Theme Switch component. All components automatically adapt to dark mode.

Automatic Theme Detection

pm7-ui automatically detects the user's system preference using prefers-color-scheme. No setup required!

Theme Switch Component

Add a theme toggle to let users switch between light and dark modes:

<!-- Basic theme switch -->
<div class="pm7-theme-switch" data-pm7-theme-switch>
  <span>Theme</span>
</div>

<!-- Small size variant -->
<div class="pm7-theme-switch pm7-theme-switch--sm" data-pm7-theme-switch></div>

Preventing Dark Mode Flicker

Important: To prevent a flash of light mode when loading dark mode, add this script to your <head> before any stylesheets:

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  
  <!-- Dark mode flicker prevention - MUST come before stylesheets -->
  <script>
    (function() {
      const savedTheme = localStorage.getItem('pm7-theme');
      if (savedTheme === 'dark' || (!savedTheme && window.matchMedia && window.matchMedia('(prefers-color-scheme: dark)').matches)) {
        document.documentElement.classList.add('dark');
      }
    })();
  </script>
  
  <!-- Your stylesheets come AFTER the script -->
  <link rel="stylesheet" href="@pm7/core/dist/pm7.css">
</head>

Manual Theme Control

You can also manually control the theme by adding/removing the dark class:

// Toggle dark mode programmatically
document.documentElement.classList.toggle('dark');

// Or set explicitly
document.documentElement.classList.add('dark');    // Enable dark mode
document.documentElement.classList.remove('dark'); // Enable light mode

Custom Dark Mode Colors

Override dark mode colors using CSS variables:

.dark {
  /* Custom dark mode colors */
  --pm7-background: #0a0a0a;
  --pm7-foreground: #fafafa;
  --pm7-primary: #3b9eff;
}

Dark Mode Best Practices

When implementing dark mode in your application, follow these best practices we've learned:

  • Use CSS variables everywhere - Never hardcode colors. Use var(--pm7-surface) instead of white or #ffffff
  • Test all pages - Dark mode should work consistently across your entire application
  • Define link colors - Don't rely on browser defaults. Set link colors explicitly: 
    a { color: var(--pm7-primary); }
.dark a { color: var(--pm7-primary); }
.dark a:visited { color: var(--pm7-primary); }
  • Check inline styles - Replace inline style="background: white" with CSS variables
  • Test component states - Verify hover, focus, active, and disabled states in both themes
  • Avoid CSS conflicts - Be careful with global selectors that might override dark mode styles
